/* CSS SPECIFICITY & CASCADE (Quick Notes) 1. CSS Specificity: -------------------- - Specificity is a scoring system used by browsers to decide which CSS rule applies if multiple rules target the same element. Priority (lowest to highest): ------------------------------ 1. Universal selector (*) or combinators (>, +, ~) — 0 points 2. Element/tag selectors (e.g., div, p, h1) — 1 point 3. Class selectors (.class), attribute selectors ([type="text"]), pseudo-classes (:hover, :nth-child) — 10 points 4. ID selectors (#id) — 100 points 5. Inline styles (style="") — 1000 points 6. !important — overrides everything (even higher specificity) 2. Cascade Algorithm (Rule Decision Order): -------------------------------------------- 1. Origin (Browser default < External/Internal CSS < Inline styles) 2. Importance (!important wins over normal rules) 3. Specificity (higher score wins) 4. Source order (last rule in CSS wins if all else is equal) Example: --------- - p → 1 - .class → 10 - #id → 100 - style="color: red;" → 1000 - .class1.class2:hover → 10 + 10 + 10 = 30 - #id.class → 100 + 10 = 110 Use lowest specificity possible for maintainable code. */
